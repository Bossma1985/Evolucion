LISTA DE COMANDOS PARA GIT

Comando Definición
ls Muestra el listado de carpetas en la que te encuentras
ls -a Muestra el listado de carpetas en la que te encuentras incluyendo archivos ocultos
pwb Muestra la carpeta en la que te encuentras
cd Te lleva a la ruta indicada
cd .. Me retorna una carpeta atrás en la gerarquía
mkdir Crea una nueva carpeta en dicho directorio (mkdir carpeta1ejemplo)
rm Borra un archivo (rm "nombrearchivo")
rm -r Borrar la carpeta (rm -r "carpeta_a_eliminar") dentro de la ruta de la carpeta
mv cambia el nombre el archivo (mv "archivo1" "archivo2") cambia del archivo1 a archivo2
cat Muestra el contenido del archivo (cat nombrearcivo) y aparece en la consola
git help Te muestra toda la ayuda
git help -a Te hace un listado completo de todos los comandos añadiendo una breve explicación
git "comando" --help Se abre una página en el navegador explicando el comando seleccionado
git status Muestra el estado de los archivos des ese repositorio (en verde: actualizados, en rojo: pendiene)
git status -s Ofrece los datos de Status de forma más reducida
git log Te muestra el registro de commits con sus comentarios y un identificador único para cada uno
git log --oneline Lo mismo que el de arriba pero de forma más esquematizada y limpia
git log --graph Fuorma alternativa de visualizar la cadena de commits realizados
git init Inicia el proyecto dentro de la carpeta donde lo ejecutas
git add Añade un archivo seleccionado al repositorio
git add . Añade todos los archivos de esa carpeta al repositorio
git branch Nos muestra la rama en la que estamos trabajando
git checkout -b Crea un rama nueva (git checkout -b nuevarama) Creará una rama llamada: nuevarama y te llevará a ella automáticamente al presionar intro
git checkout Seguido del nombre de la rama a la que quieras acceder te llevará a ella sin crear una nueva / También te permite navegar entre los distintos commits (git checkout "identificador commit")
git merge Incluye los cambios de una subrama a la rama principal. Debes estar dentro de la rama principal para que surja efecto (git merge "nombredelarama")
git diff Te muestra un resumen de los cambios (1º Se aplica el comando, 2º se hacen cambios en los archivos y 3º se vuelve a activar el comando para ver la lista de cambios
code . Abre VS Code con los archivos de ese directorio
git commit -m "" Hace una especie de fotografía del estado del proyecto en ese momento. Es indispensable escribir un comentario entre las comillas
git commit Lo mismo que el de arriba pero te lo abre en VS Code (añadir comentario, guardar y cerrar VS Code)
git push Sube los cambios hechos en el repositorio a un servidor en la nube (GitHub)
.gitignore Se crea un archivo .gitignore y se guarda el el repositorio, una vez hecho esto solo hay que escribir dentro de dicho archivo los nombres de los archivos que no quieres que aparezcan y si solo escribes la extensión ".txt, .html, .js…" se ignoran todos los de esa extensión. Para ignorar una carpeta hay que escribir su ruta con el / final "carpeta_fotos/"
git remote add origin Seguido del link que me muestra dentro del repositorio en Git Bash, le dice a git en que repositorio debe subir los cambios realizados. Es como el enlace de comunicación entre local y remoto
git remote rm origin Te devuelve al estado anterior
git push -u origin main Push lo sube, -u crea la rama en GitHub, origin es la dirección y master el nombre que le damos
git reset "Elimina" todos los commits a partir del identificador que le señales (es una operación de riesgo aunque es revertible
